How this works

We shuffle the player input (as if it was a bunch of names written on index cards) and assign the "top of the deck" to one prize at a time. The prizes aren't shuffled, so they stay in the order that you put them in. This could be helpful if you put the "grand prizes" at the top, or the bottom if you like drama. Prizes are assigned in order to the names in the "deck" until we run out of prizes.

All data resides in your web browser; the data is never uploaded to the server. So, "the server" never has it.

If you decide to use this for a random drawing, it's a good idea to have a witness, give a demo of the program, then reset the results, and click "select winners" one time "for real"--and cut-and-paste into a document for permanent storage.

About the shuffle

This version uses your browser's JavaScript's built-in Math.random() and the unfortunate realities of JavaScript IEEE-754 floating point. That's OK for trivial things but I wouldn't recommend it for running your state's lottery.

Specifically, we probably introduce a trivial bias by multiplying using the output of Math.random(); we have not verified the implementation of your browser's Math.random(); and the results are not publically verifyable or repeatable because we do not know the state of your browser's PRNG.

The names are shuffled twice. One shuffle is a Fisher-Yates shuffle. This is known to be generally fair if the RNG is fair, unless there are bugs. The second shuffle is a randomized-sort. This is known to have a small bias depending on how likely it is that the RNG picks the same item, but Picky's implementation is less likely to have bugs. Combined, they [should] do a good [enough] job.
